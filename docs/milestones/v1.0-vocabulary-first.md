# Milestone 1.0 — Vocabulary-first dense fill

Date: 2026-07-18

## Decision

Vocabulary-first dense fill is promoted from research to the default generator pipeline.

The production path is:

```text
39,586-entry attributed source corpus
-> deterministic length-balanced working sets at 2,500 and 3,500 entries
-> full construction and clue allocation for both candidates
-> same-geometry editorial repair for each candidate
-> panel-first candidate selection
-> complete structural validation
```

The previous small-dictionary generator remains reproducible with `SCANWORD_BULK_LEXICON=off`. The vocabulary portfolio and editorial repair can be disabled independently for rollback and A/B analysis.

## Production scope

Promoted components:

- the 39,586-entry pre-construction corpus;
- generated browser/Node corpus loader;
- length-balanced seed-specific pool selection;
- 2,500/3,500 active-set vocabulary portfolio;
- same-geometry single, pair and radius-two editorial repair;
- complete validation of runs, crossings, connectivity and exact clues;
- dictionary audit and milestone CI.

Not promoted as default:

- category caps for cities, names and specialist terms;
- lexical placement penalties;
- pre-downstream lexical Pareto selection;
- raw full-corpus greedy sampling;
- experimental closed-fill CSP variants as independent release claims.

Those experiments remain documented in the research dossiers and branch history.

## Experiment record

### 1. Same-geometry repair

The unified repair pipeline proved that formulaic short answers can be removed without changing slot geometry. It did not reduce structural short-slot count or residual panels, so it remains a final cleanup stage.

### 2. Naive full-pool construction — rejected

Exposing the full 22,500-entry precursor corpus directly to the existing greedy constructor buried short bridge words, increased panels and slowed generation.

Conclusion: source-corpus size and per-generation working-set size are separate controls.

### 3. Length-balanced 5,000-entry set — partial success

The balanced set restored answer and crossing gains but retained a small panel regression. This validated the sampling architecture while leaving the active-set size unresolved.

### 4. Active-set sweep

The 2,500 / 3,500 / 5,000 / 7,500 sweep showed that no single fixed limit dominated every seed. The 3,500-entry set was initially selected as the stable default; the 2,500-entry set later showed stronger answer density but higher short-fill cost.

### 5. 39,586-entry corpus

The expanded corpus added countries, doubled geography and proper names, and increased noun coverage to the current source ceiling under the filters.

A fixed 3,500-entry working set improved the original 887-word construction baseline over 20 seeds:

| metric | original | vocabulary-first 3,500 |
| --- | ---: | ---: |
| panels | 7.35 | 6.20 |
| answers | 44.90 | 47.35 |
| crossings | 47.20 | 52.00 |
| raw-letter coverage | 49.21% | 51.55% |
| formulaic short answers | 4.00 | 1.55 |

### 6. Category caps — retained experiment, not default

Soft caps reduced some proper-name usage and increased answer count, but did not consistently reduce cities and worsened the unrepaired short-fill penalty. Repair removed the formulaic tail, but the tested cap profile still did not dominate the uncapped source selection.

### 7. Vocabulary portfolio — promoted

The final release candidate constructs and repairs both 2,500- and 3,500-entry candidates, then selects panel-first.

Validated over 20 identical-seed pairs against the old dictionary with the same editorial repair:

## Accepted trade-offs

The milestone intentionally accepts:

- an average generation time near 21 seconds for the default 13 × 17 benchmark;
- a higher aggregate editorial penalty than the small-dictionary repaired baseline because the denser grids use more legitimate short bridge answers;
- substantial generic-clue debt for names and geography;
- occasional panel or answer regression on individual seeds, bounded by portfolio selection and complete validation.

The milestone does not claim zero-panel generation or publication-ready clue prose.

## Rollback controls

- `SCANWORD_BULK_LEXICON=off` — old construction dictionary;
- `SCANWORD_VOCABULARY_PORTFOLIO=off` — one working set only;
- `SCANWORD_EDITORIAL_REPAIR=off` — no final same-geometry cleanup;
- `SCANWORD_CATEGORY_BALANCE=off` — current accepted category policy;
- `SCANWORD_CONSTRUCTION_MODE=legacy` — original construction path.
