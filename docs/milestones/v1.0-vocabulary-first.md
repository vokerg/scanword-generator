# Milestone 1.0 — Vocabulary-first dense fill

Date: 2026-07-18

Status: historical milestone; superseded by [Vocabulary-first 1.1](v1.1-vocabulary-greatness.md).

## Decision

Vocabulary-first dense fill was promoted from research to the default generator pipeline.

The 1.0 path was:

```text
39,586-entry attributed source corpus
-> deterministic length-balanced working sets at 2,500 and 3,500 entries
-> full construction and clue allocation for both candidates
-> same-geometry editorial repair for each candidate
-> panel-first candidate selection
-> complete structural validation
```

The previous small-dictionary generator remained reproducible with `SCANWORD_BULK_LEXICON=off`. The vocabulary portfolio and editorial repair could be disabled independently for rollback and A/B analysis.

## Production scope at 1.0

Promoted components:

- the 39,586-entry pre-construction corpus;
- generated browser/Node corpus loader;
- length-balanced seed-specific pool selection;
- 2,500/3,500 active-set vocabulary portfolio;
- same-geometry single, pair and radius-two editorial repair;
- complete validation of runs, crossings, connectivity and exact clues;
- dictionary audit and milestone CI.

Not promoted as default:

- category caps for cities, names and specialist terms;
- lexical placement penalties;
- pre-downstream lexical Pareto selection;
- raw full-corpus greedy sampling;
- experimental closed-fill CSP variants as independent release claims.

## Corpus checkpoint

The source corpus contained 39,586 unique clue-bearing entries:

| category | entries |
| --- | ---: |
| common nouns | 4,358 |
| specialist nouns | 20,099 |
| given names | 2,798 |
| surnames | 2,087 |
| patronymics | 115 |
| cities | 9,830 |
| capitals | 170 |
| countries | 129 |

Audit result:

- invalid entries: 0;
- duplicate normalized answers: 0;
- entries with clues: 100%;
- source attribution retained: 100%;
- template clues: 15,136, or 38.24%.

The template share was accepted as explicit editorial debt.

## Experiment record

### Same-geometry repair

The unified repair pipeline proved that formulaic short answers could be removed without changing slot geometry. It did not reduce structural short-slot count or residual panels, so it remained a final cleanup stage.

### Naive full-pool construction — rejected

Exposing the full 22,500-entry precursor corpus directly to the existing greedy constructor buried short bridge words, increased panels and slowed generation. Source-corpus size and per-generation working-set size therefore remained separate controls.

### Length-balanced 5,000-entry set — partial success

The balanced set restored answer and crossing gains but retained a small panel regression. This validated the sampling architecture while leaving active-set size unresolved.

### Active-set sweep

The 2,500 / 3,500 / 5,000 / 7,500 sweep showed that no fixed limit dominated every seed. The final solution was a bounded portfolio.

### Category caps — retained experiment, not default

Soft caps reduced some proper-name usage and increased answer count, but did not consistently reduce cities and worsened unrepaired short-fill cost. The tested cap profile did not dominate the uncapped source selection.

### Vocabulary portfolio — promoted

Validated over 20 identical-seed pairs against the old dictionary with the same editorial repair:

| metric | old dictionary + repair | vocabulary portfolio + repair | delta |
| --- | ---: | ---: | ---: |
| average residual panels | 7.05 | 5.30 | -1.75 |
| average answers | 44.75 | 48.45 | +3.70 |
| average crossings | 47.35 | 53.00 | +5.65 |
| average answer-space coverage | 93.98% | 95.53% | +1.55 pp |
| average raw-letter coverage | 49.50% | 51.03% | +1.54 pp |
| average formulaic short answers | 0.40 | 0.15 | -0.25 |

Outcome counts:

- fewer panels: 15/20;
- more answers: 17/20;
- more crossings: 18/20;
- panel regressions: 3/20;
- answer regressions: 1/20;
- complete structural validation: 20/20;
- selected active limit: 2,500 on 14 seeds, 3,500 on 6 seeds.

## Runtime correction recorded by 1.1

The original 1.0 release report listed an average portfolio runtime of 21.11 seconds. A later browser-equivalent Node bootstrap showed that the benchmark harness had omitted final runtime modules. The reproduced 20-seed result preserved every structural metric above but measured:

- former dictionary: 10.86 seconds;
- vocabulary portfolio: 24.48 seconds.

Historical metrics are not silently rewritten; this section records the correction.

## Accepted trade-offs at 1.0

The milestone accepted:

- two complete candidate evaluations;
- higher short-answer editorial cost than the small dictionary on some seeds;
- substantial generic-clue debt;
- occasional per-seed panel or answer regressions;
- no claim of zero-panel generation or publication-ready clue prose.

## Rollback controls

- `SCANWORD_BULK_LEXICON=off` — old construction dictionary;
- `SCANWORD_VOCABULARY_PORTFOLIO=off` — one working set only;
- `SCANWORD_EDITORIAL_REPAIR=off` — no final same-geometry cleanup;
- `SCANWORD_CATEGORY_BALANCE=off` — accepted category policy;
- `SCANWORD_CONSTRUCTION_MODE=legacy` — original construction path.
